# Kitty Cloud 更新日志

## 2026-05-20 聊天功能与模型切换

### 新增功能

#### 1. 文字聊天界面 (ChatView)

- 独立的文字聊天页面，与语音通话并行
- 消息气泡式布局，用户（蓝色）和 Kitty（绿色）分列两侧
- 模拟打字动画效果，emoji 整体显示不拆分
- 打字指示器（三点脉动动画）
- 错误提示、清空对话确认

#### 2. 模型切换

- 通过 `/models` 接口从 OpenClaw WS RPC 动态获取模型列表
- 聊天页面和语音通话页面均可切换模型
- 模型列表带 5 分钟缓存，避免频繁请求
- 切换后立即生效，后续聊天和语音通话都使用新模型

#### 3. 模型切换实现原理

OpenClaw HTTP API 的模型切换通过 `x-openclaw-model` header 实现：

```
客户端选择模型 (如 qianfan/deepseek-v4-flash)
    ↓
Kitty Server 收到 model 参数
    ↓
通过 x-openclaw-model header 转发给 OpenClaw
    ↓
OpenClaw 路由到对应底层模型
```

**注意**：OpenClaw HTTP API 的 `model` 字段只接受 `openclaw` 或 `openclaw/<agentId>` 格式，不支持 `provider/model` 格式。模型切换必须通过 header。

### 修复问题

#### 1. 聊天返回空内容

- **原因**：客户端传的 `provider/model` 格式模型名被直接转发给 OpenClaw HTTP API，该 API 不接受此格式，返回 `Invalid model` 错误
- **修复**：HTTP 请求的 `model` 字段固定为 `openclaw/main`，模型切换通过 `x-openclaw-model` header

#### 2. SSE 格式不正确

- **原因**：原服务端返回纯文本流，客户端期望 SSE 格式
- **修复**：`Content-Type` 改为 `text/event-stream`，内容包装为 `data: {json}\n\n` 格式

#### 3. 非流式模式支持

- **原因**：代理缓冲导致流式响应被延迟
- **修复**：新增 `stream` 参数，支持非流式模式返回完整 JSON 响应

#### 4. 图标显示为问号

- **原因**：emoji 字符在部分 iOS 渲染环境下显示异常
- **修复**：所有 emoji 图标替换为 SF Symbols（`cat.fill` = Kitty，`crown.fill` = 老板）

#### 5. 模型显示名不更新

- **原因**：`availableModels` 为空时回退到硬编码 "Qwen3.6 Plus"
- **修复**：fallback 改为从 model id 提取显示名，页面出现时自动拉取模型列表

### 系统提示词

统一语音和文字聊天的系统提示词，强调：
- 口语化表达，不用书面语
- 禁止 Markdown 格式（**、*、#、-、` 等）
- 禁止 emoji 表情
- 纯文本输出

### 技术细节

| 项目 | 实现 |
|------|------|
| 聊天模式 | 非流式（chatComplete）+ 模拟打字动画 |
| 语音模式 | 流式 SSE（chatStream）+ 实时播放 |
| 模型切换 | x-openclaw-model header |
| 模型列表 | OpenClaw WS RPC models.list + 5min 缓存 |
| 图标 | SF Symbols (cat.fill, crown.fill) |
| 打字动画 | Character (grapheme cluster) 逐字追加，emoji 整体显示 |

## 2026-05-19 初始聊天功能

### 服务端 SSE 格式改造

- `/chat` 接口返回格式从 `text/plain` 改为 `text/event-stream`
- 响应内容包装为 SSE 格式：`data: {"choices": [{"delta": {"content": "..."}}]}\n\n`
- 添加结束标志：`data: [DONE]\n\n`

### 客户端动态省略号效果

- `TypingIndicatorView`：三个圆点的脉动动画
- 绿色配色，与 Kitty 主题一致
