### 3.2 通话中的核心能力

#### 3.2.1 实时对话体验

- **延迟**：端到端 ≤ 800ms（优于人脑反应时间感知极限）
- **打断机制**：用户可随时说话打断AI，AI立即停止当前回复并倾听
- **静音检测**：用户静默3秒后，AI可主动询问“还在听吗？”
- **背景降噪**：自动抑制键盘、风声、交通噪音（移动端实时）

#### 3.2.2 对话状态与情绪感知

- AI能识别用户**情绪倾向**（高兴/低落/焦虑/愤怒）并调整语气
- 通话中**不主动挂断**，除非用户手动挂断或网络中断超过20秒
- 支持**中途静音自己**（对AI说话，但AI听不到用户）

#### 3.2.3 通话控制条（悬浮 / 固定）

- 切换扬声器/听筒

## 六、异常与边界场景处理

| 场景                      | 处理方式                                                 |
| :------------------------ | :------------------------------------------------------- |
| 网络从WiFi切到蜂窝        | 无缝切换，不挂断                                         |
| 通话中接听电话            | 自动暂停对话，等电话挂断后恢复（提示“刚才说到哪里了？”） |
| 用户长时间不说话（>60秒） | AI提示“我要先挂断了，需要随时呼叫我”，10秒后挂断         |
| AI回复过慢（>3秒）        | 播放轻量缓冲音效（类似“嗯…让我想想”）                    |
| 用户多次打断AI            | AI缩短回复长度，变得更简洁                               |

其他：

* 必须“短句优先”

* 可能要在程序中准备一个提示词框架，让 AI 一直知道是在跟人对话。

* AI不要：

  - 一口气说 10 秒
  - 没有停顿

  要做到：

  - 2~4 秒一小段

  - 中间可被打断
